# v0.1 Smoke Test Checklist

Прогнать руками после имплементации. Все шаги — на dev-машине, против реальных сервисов.

## Подготовка
- [ ] `node --version` или `bun --version` печатает версию
- [ ] `~/.claude/projects/` существует и содержит JSONL логи
- [ ] (опционально) `~/.codex/` существует
- [ ] Создан GitHub PAT c `repo` + `read:user`

## First launch
- [ ] Запуск собранного `.app` — иконка появляется в menu bar
- [ ] Алёрт «Config initialized» появился
- [ ] `~/.config/ai-stats/config.json` создан с пустыми токенами
- [ ] Закрыть алёрт, заполнить токен, перезапустить

## Sync
- [ ] В течение 30 секунд после старта `stats.db` появляется в `~/Library/Application Support/ai-stats/`
- [ ] `sqlite3 stats.db "SELECT source, COUNT(*) FROM ai_usage GROUP BY source"` — есть строки для `claude` и/или `codex`
- [ ] `sqlite3 stats.db "SELECT COUNT(*) FROM github_activity"` > 0
- [ ] `sqlite3 stats.db "SELECT * FROM sync_state"` — записи без `last_error`

## UI
- [ ] Клик по иконке — popover открывается, цифры не нули
- [ ] Переключение Day/Week/Month — числа меняются
- [ ] Sparkline нарисован
- [ ] `$X.XX` в menu bar совпадает с цифрой в popover при выбранном Day
- [ ] Кнопка ⚙ открывает Settings window

## Settings
- [ ] Export → выбираешь файл `.db` → файл создаётся, открывается через `sqlite3` без ошибок
- [ ] Удалить созданный backup-файл вручную для следующего теста
- [ ] Import → выбираешь свой же экспорт → алёрт подтверждения → импорт проходит → app завершается → запускаешь заново → данные те же
- [ ] В `~/Library/Application Support/ai-stats/` лежит `stats.db.backup-<timestamp>`

## Never-decrease
- [ ] `mv ~/.claude/projects ~/.claude/projects.hidden`
- [ ] Нажать Refresh now в Settings (или подождать 5 мин)
- [ ] Открыть popover — цифры **не уменьшились**
- [ ] `mv ~/.claude/projects.hidden ~/.claude/projects` (вернуть)

## После прогона
- [ ] Все галочки выше — стоят
- [ ] Багов нет, или баги — отдельными `fix(...)` коммитами
- [ ] Tag релиза: `git tag -a v0.1.0 -m "v0.1.0 — personal MVP"`
